1. 产品简介

BiliBili弹幕Claw 是一款 B站直播间弹幕监听 + AI自动回复 桌面工具。 它能实时接收直播间弹幕，按规则过滤后交给大语言模型（LLM）生成回复，并以弹幕形式自动发送到直播间。

核心功能

  • 实时弹幕监听 — 通过 B站 WebSocket 协议接收弹幕、礼物、醒目留言
  • 关键词/正则匹配 — 灵活配置过滤规则，支持匹配范围选择
  • 固定回复 — 对匹配弹幕直接发送预设文本，不走 AI，响应速度更快
  • AI 自动回复 — 接入 OpenAI 兼容接口，上下文感知地生成弹幕回复
  • 粉丝牌等级过滤 — 设置最低粉丝牌等级，过滤低等级弹幕
  • 礼物 & 醒目留言识别 — 自动将礼物/SC 事件送入 AI，实现感谢互动
  • 防自回环 — 回声检测机制避免机器人与自身弹幕死循环
  • 配置加密存储 & 导出/导入 — Cookie 等敏感信息加密保存，支持一键迁移

2. 系统架构与数据流

B站直播间 │ WebSocket ▼ Python 弹幕核心 (blivedm) │ JSON-RPC over stdio ▼ Electron 主进程 ├── DanmakuService ── 关键词过滤 ── 作用域路由 │ │ │ ├──► 固定回复引擎 (QuickReplyEngine) │ │ 匹配 → 直接发弹幕 │ │ │ └──► AI 回复引擎 (AIRelayManager) │ 排队 → 调用 LLM → 解析回复 → 发弹幕 │ │ │ 命中跳过标记 → 不发送 │ └──► Renderer 进程 (Vue 3 GUI) └── 实时显示源弹幕 / 匹配弹幕 / AI 状态

弹幕路由逻辑

当一条弹幕到达时，系统按以下流程处理：

  1. 粉丝牌等级过滤：低于门槛的弹幕不会进入匹配/AI，但源弹幕仍正常显示。
  2. 关键词匹配：弹幕与关键词/正则规则进行匹配，并得到 scope（匹配范围）。
  3. 作用域路由：
    • 全部匹配 — 匹配的弹幕同时进入固定回复和 AI。
    • 仅固定回复 — 匹配的弹幕只进固定回复；未命中的弹幕直接进 AI（绕过过滤）。
    • 仅AI回复 — 匹配的弹幕只进 AI；未命中的弹幕直接进固定回复（绕过过滤）。
  4. 固定回复优先：如果命中固定回复规则，直接发送，不再进入 AI 队列。
  5. AI 回复：进入队列，按节奏调用大模型，解析回复文本后发送弹幕。

3. 快速开始

3.1 安装

  1. 双击 BiliBili弹幕Claw Setup 0.1.0.exe 运行安装向导。
  2. 选择安装目录，完成安装。
  3. 启动 BiliBili弹幕Claw。
注意
首次启动可能被 Windows Defender 拦截，请选择"仍要运行"。Python 弹幕核心已打包为独立可执行文件，无需单独安装 Python。

3.2 首次运行

  1. 获取 B站凭证 — 在「直播间」页点击"弹出B站登录页"，扫码登录后自动获取 Cookie；或手动从浏览器开发者工具中复制 SESSDATA、bili_jct、buvid3。
  2. 配置关键词 — 在「关键词」页添加你想关注的弹幕关键词或正则。
  3. 配置大模型 — 在「大模型配置」页选择供应商和模型，填入 API Key，编辑提示词。
  4. 开始监听 — 回到「直播间」页输入房间号，点击"开始监听"。

4. 直播间页面

4.1 连接设置

配置项说明
房间号B站直播间短号或真实房间号
开始监听连接直播间 WebSocket，开始接收弹幕
停止监听断开连接，可选发送告别弹幕
断开前发送告别弹幕勾选后，停止监听前自动发送告别文案
告别文案停止监听时发送的弹幕内容，如"先下播啦，感谢大家～"

4.2 B站凭证

程序需要 B站 Cookie 中的三个字段来连接弹幕服务器和发送弹幕：

字段必填说明
SESSDATA是登录凭证，用于身份验证
bili_jct是CSRF Token，用于发送弹幕等写操作
buvid3可选设备标识，部分接口需要
推荐方式
点击"弹出B站登录页"，在弹出的浏览器窗口中扫码或登录，程序会自动抓取并保存 Cookie。无需手动复制粘贴。

4.3 粉丝牌等级过滤

在直播间页面可设置「最低粉丝牌等级」：

  • 设为 0 表示不过滤。
  • 设为 N 表示只让粉丝牌等级 ≥ N 的弹幕进入关键词匹配和 AI 自动回复。
  • 源弹幕不受影响，仍会正常显示在监控列表中。

5. 关键词页面

5.1 关键词规则

关键词规则决定哪些弹幕被"捕获"进入后续处理。每条规则包含：

属性说明
类型关键词 子串包含匹配，或 正则 JavaScript 正则表达式匹配
模式具体匹配的文本或正则表达式
区分大小写仅关键词类型有效；正则通过 flags 自行控制
匹配范围见 5.2 节
启用可暂停某条规则而不删除

5.2 匹配范围

每条关键词规则可以选择三种作用域：

范围标签匹配弹幕处理未匹配弹幕处理
全部匹配 ALL 同时进入固定回复和 AI 不进入任何处理
仅固定回复 固定 只进固定回复，不进 AI 直接进 AI（绕过过滤）
仅AI回复 AI 只进 AI，不进固定回复 直接进固定回复（绕过过滤）
理解作用域

「仅固定回复」意味着关键词只影响固定回复的过滤，AI 不受这条规则限制，会直接读源弹幕。这适合你想让 AI 对所有弹幕都能回复，但只对特定关键词触发的弹幕做固定回复的场景。

「仅AI回复」则相反：关键词只控制 AI 的过滤，固定回复不受这条规则限制。

5.3 固定回复规则

固定回复（Quick Reply）是独立于 AI 的即时回复机制，优先级高于 AI。当弹幕命中固定回复规则时，直接发送预设文本，不再进入 AI 队列。

规则字段

字段说明
包含词弹幕包含其中任一词语即触发（逗号或换行分隔）
排除词弹幕包含其中任一词语则不触发
正则填写后优先使用正则匹配，可使用捕获组
回复内容命中后发送的弹幕文本
冷却（秒）同一规则两次触发之间的最小间隔，0 表示无冷却
区分大小写是否对包含词/正则做大小写敏感匹配
启用可暂停规则而不删除
示例

包含词：什么游戏, 啥游戏，排除词：货物，回复内容：今晚在玩原神！，冷却：30 秒。那么弹幕"什么游戏啊"会触发回复，但"什么游戏货物便宜"不会触发（因为包含"货物"）。30 秒内不会重复触发。

固定回复与关键词范围
固定回复的弹幕是否需要先通过关键词过滤，取决于关键词规则的匹配范围设置。如果所有关键词范围都是「仅AI回复」，那么固定回复会直接处理所有源弹幕，不受关键词限制。

5.4 粉丝牌等级门槛

在关键词页面也可以设置粉丝牌等级门槛（与直播间页面同步）。设置后，粉丝牌等级低于门槛的弹幕不会进入关键词匹配和 AI 自动回复。

5.5 忽略用户名

在「忽略用户名」区域，填写不想让 AI 回复的用户名（每行一个或逗号分隔）。这些用户发送的弹幕仍会显示在源弹幕列表中，但不会进入 AI 回复队列，避免机器人自回环。

6. 大模型配置页面

6.1 供应商与模型

目前支持 OpenCode 统一 API 接入，可选用 GPT 系列、Claude 系列、Gemini 系列、MiniMax、GLM、Kimi 等模型。

配置项说明
供应商选择 API 供应商（当前仅 OpenCode）
模型从下拉列表选择模型，会自动填充对应的 Endpoint
API Key从供应商控制台获取的密钥
Endpoint根据模型自动选择，一般无需手动修改
发送间隔(ms)两条弹幕之间的最小发送间隔，默认 1800ms，最低 500ms
队列上限待发送弹幕缓冲队列最大长度，默认 100，超限后仍继续排队
注意
「保存供应商配置」只保存配置，不会连接模型。配置保存后需在下方「提示词配置」区域点击"开始连接"才会建立 AI 会话。

6.2 提示词

系统提示词（System Prompt）定义了 AI 的角色和行为。编辑后会自动保存到配置文件。

推荐在提示词中约定输出格式：

提示词约定

告诉模型：需要回复时输出弹幕文本；不需要回复时仅输出 NO_REPLY。这样系统会自动跳过不需要回复的消息，避免刷屏。

默认提示词为：「你现在是一个直播间助理，你会收到粉丝牌+用户名+弹幕内容，请逐条回复，单条回复不超过40字。」

6.3 跳过标记

在「无需发送标记」区域可配置跳过标记，每行一个。当模型输出命中这些标记时（完全匹配或前缀匹配），本条消息将跳过发送。

默认跳过标记：

  • NO_REPLY
  • 无需回复
  • 不需要回复
  • 不用回复
  • 不回复
  • 忽略
  • skip
  • pass
匹配规则
跳过判断为大小写不敏感的前缀匹配，支持 NO_REPLY、NO_REPLY: 原因、[NO_REPLY]、【无需回复】 等格式。

6.4 AI返回预览

连接 AI 后，页面底部会实时显示最近 50 条 AI 决策记录，包括：

字段说明
入参发送给模型的输入文本（包含粉丝牌、用户名、弹幕）
输出模型返回的文本（截断到40字）
状态待发送 / 已发送 / 已跳过 / 错误 / 已清空
原因跳过原因或发送详情

可点「复制」查看单条详情，或点「复制全部预览」导出所有记录。

7. 配置导入/导出

所有配置（房间号、Cookie、关键词、固定回复、AI 模型设置等）均支持导出和导入，方便在不同电脑间迁移。

7.1 导出配置

在「设置」页面点击「导出配置」，选择保存路径，程序会将当前所有配置导出为明文 JSON 文件，包括：

  • 房间配置（房间号、最低粉丝牌等级、断开消息等）
  • B站凭证（SESSDATA、bili_jct、buvid3）
  • 关键词规则
  • 固定回复规则
  • AI 模型配置（供应商、API Key、提示词、跳过标记等）
安全提醒
导出的 JSON 文件包含 B站 Cookie 和 API Key 等敏感信息，请妥善保管，不要公开分享。

7.2 导入配置

在「设置」页面点击「导入配置」，选择之前导出的 JSON 文件。程序会自动读取并覆盖当前配置。

导入后，以下内容会立即生效：

  • 关键词规则 → 更新到弹幕服务
  • 固定回复规则 → 更新到固定回复引擎
  • 粉丝牌等级门槛 → 更新到弹幕服务
注意
AI 模型连接和直播间连接需要手动重新连接。导入配置不会自动开始监听或连接 AI。

8. 常见问题

Q: 连接直播间后没有弹幕？

A: 请检查：

  • 房间号是否正确
  • SESSDATA 和 bili_jct 是否有效（过期需重新获取）
  • 直播是否正在进行
  • 查看控制台（如果可打开 DevTools）是否有错误日志

Q: AI 不回复？

A: 请检查：

  • 是否已点击「大模型配置」页的"开始连接"
  • API Key 是否正确
  • 模型 Endpoint 是否可访问
  • 弹幕是否命中了关键词/粉丝牌过滤
  • 查看「AI返回预览」是否有"已跳过"记录（可能命中了跳过标记）

Q: 弹幕发送失败？

A: 可能原因：

  • bili_jct 过期或错误
  • 发送间隔太短（B站有频率限制，建议不低于1800ms）
  • 弹幕内容超长（B站限制约20-30字，本程序截断到40字）

Q: 如何避免 AI 回复自己？

A: 在「关键词」页面底部「忽略用户名」中填入你直播间的昵称或机器人账号，这些用户发送的弹幕不会进入 AI 回复队列。

Q: 固定回复和 AI 回复哪个优先？

A: 固定回复始终优先于 AI。如果弹幕同时命中了固定回复规则和关键词规则，只会发送固定回复，不会进入 AI 队列。

Q: 礼物/醒目留言能触发 AI 回复吗？

A: 可以。礼物和醒目留言会自动作为特殊事件送入 AI 队列，格式为"🎁 用户名 送出 礼物名 x数量"。AI 会根据提示词决定是否回复感谢。

Q: 导入配置后固定回复规则没有加载？

A: v0.1.0+ 已修复此问题，导入配置时固定回复规则（quickReplies）会一并加载。如果使用较早版本导出的配置，可能需要手动添加固定回复规则。
